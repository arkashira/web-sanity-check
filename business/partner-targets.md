# Partner Targets
## Overview
As a comprehensive web application testing tool, web-sanity-check aims to integrate with various SaaS and APIs to expand its capabilities and reach a broader audience. This document outlines the partner targets, integration roadmap, and specific SaaS/APIs to integrate.

## Partner Integration Roadmap
### Phase 1: Core Integrations (Q3 2026)
1. **New Relic**: Integration effort: M (Medium)
	* Value-add: Solves the problem of monitoring and troubleshooting web applications for DevOps engineers.
	* Affiliate/revenue-share: Yes (10% revenue share)
	* Free-tier limits: 100,000 transactions per month
2. **Sentry**: Integration effort: S (Small)
	* Value-add: Helps developers identify and fix errors in web applications.
	* Affiliate/revenue-share: Yes (15% revenue share)
	* Free-tier limits: 10,000 errors per month
3. **AWS CodePipeline**: Integration effort: M (Medium)
	* Value-add: Automates testing and deployment of web applications on AWS.
	* Affiliate/revenue-share: No
	* Free-tier limits: 100,000 pipeline runs per month

### Phase 2: Extended Integrations (Q4 2026)
1. **Postman**: Integration effort: L (Large)
	* Value-add: Enhances API testing capabilities for web applications.
	* Affiliate/revenue-share: Yes (12% revenue share)
	* Free-tier limits: 10,000 requests per month
2. **CircleCI**: Integration effort: M (Medium)
	* Value-add: Automates testing and deployment of web applications on CircleCI.
	* Affiliate/revenue-share: No
	* Free-tier limits: 100,000 builds per month
3. **DigitalOcean**: Integration effort: M (Medium)
	* Value-add: Provides scalable infrastructure for web applications.
	* Affiliate/revenue-share: Yes (8% revenue share)
	* Free-tier limits: 10,000 hours of usage per month

### Phase 3: Strategic Partnerships (Q1 2027)
1. **Google Cloud**: Integration effort: L (Large)
	* Value-add: Enhances cloud-based testing and deployment capabilities for web applications.
	* Affiliate/revenue-share: Yes (15% revenue share)
	* Free-tier limits: 100,000 hours of usage per month
2. **Microsoft Azure**: Integration effort: L (Large)
	* Value-add: Provides scalable infrastructure and testing capabilities for web applications.
	* Affiliate/revenue-share: Yes (12% revenue share)
	* Free-tier limits: 100,000 hours of usage per month

## Rationale
The selected SaaS/APIs are chosen based on their relevance to the web application testing space, market demand, and potential revenue opportunities. The integration effort is estimated based on the complexity of the integration and the resources required. The value-add is determined by the specific user job that the integration solves, and the affiliate/revenue-share is considered for partnerships with significant revenue potential.
